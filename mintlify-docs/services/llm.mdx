## Overview

The LLM Service powers the Judge's evaluation engine. It takes competing worker submissions and produces a structured `JudgeEvaluation` that determines the winner.

Two implementations are available:
- **`LLMService`** — Calls the remote LLM API
- **`MockLLMService`** — Deterministic algorithmic evaluation

## Evaluation Interface

```typescript
interface JudgeEvaluation {
  winnerId: string;                    // Must match a submitted workerId
  reason: string;                      // Human-readable explanation
  confidence: "HIGH" | "MEDIUM" | "LOW";  // Evaluation certainty
}
```

## Real Implementation (`LLMService`)

### Initialization

```typescript
const llm = new LLMService(
  process.env.LLM_API_KEY,
  "default-model-name"  // optional
);
```

### Evaluation Strategies

The service supports two primary evaluation modes defined in the bounty:

#### 1. Quality Mode (Default)
Prioritizes data integrity over cost.
- **Criteria 1**: More price sources is better (e.g., 3 sources > 2).
- **Criteria 2**: Lower inter-source variance is better (consensus).
- **Criteria 3**: If tied, the worker with more sources wins.

#### 2. Price Mode
Prioritizes cost-efficiency for the Requester.
- **Criteria 1**: Lower bid amount wins (cheapest task execution).
- **Criteria 2**: If tied on bid, quality (sources/variance) acts as a tiebreaker.

### Prompt Design

The prompt dynamically adapts based on the requested strategy:

```
You are an impartial judge evaluating submissions from AI worker agents.

TASK: {description} (ID: {taskId})
STRATEGY: {strategy}

SUBMISSIONS:
Worker 1: ID="0.0.WORKER_1"
  Sources: coingecko, kraken, binance
  Prices: 67150, 67155.3, 67148.2
  Average: 67151.17
  Bid Amount: 50 HBAR

{criteria_by_strategy}

Respond ONLY with this JSON object:
{"winnerId": "<workerId>", "reason": "<sentence>", "confidence": "HIGH|MEDIUM|LOW"}
```

### Mock Implementation (`MockLLMService`)

The algorithmic evaluator mirrors the LLM's logic using a deterministic sorting algorithm:

```typescript
if (effectiveStrategy === "price") {
  // Lowest bid first, then quality as tiebreaker
  scored.sort((a, b) => {
    if (a.bidAmount !== b.bidAmount) return a.bidAmount - b.bidAmount;
    if (b.sourcesCount !== a.sourcesCount) return b.sourcesCount - a.sourcesCount;
    return a.variance - b.variance;
  });
} else {
  // Quality mode: sources DESC, then variance ASC
  scored.sort((a, b) => {
    if (b.sourcesCount !== a.sourcesCount) return b.sourcesCount - a.sourcesCount;
    return a.variance - b.variance;
  });
}
```

### Why Variance Matters

| Scenario | Worker A | Worker B | Winner |
|---|---|---|---|
| Different source counts | 3 sources | 2 sources | **A** (more sources) |
| Same sources, different variance | \[45000, 45001\] (var: 0.25) | \[45000, 45100\] (var: 2500) | **A** (lower variance) |
| Same everything | \[45000\] | \[45000\] | **A** (first entry) |

Low variance between price sources indicates that the data is **more trustworthy** — the sources agree with each other.
